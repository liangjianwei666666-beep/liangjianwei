# 视觉笔记信息图

## 核心定位

把复杂理念视觉综合为简单的视觉隐喻，生成中文竖版视觉笔记信息图。画面应像一张有组织的模块化知识地图：干净留白、层级流动、视觉引导明确，兼具专业感和趣味感。

默认图像规范：

- 尺寸：1242x1660 像素，竖版高清。
- 风格：手绘扁平矢量插画，极简主义涂鸦线条艺术，视觉引导风格。
- 美学：柔和且独特的强调色，清晰、轻盈、有认知图解感。
- 文字：中文，短标签优先，字体指定为 Ma Shan Zheng。
- 布局：模块化知识地图，干净留白，层级流动，不拥挤。
- 底部标记：默认 `@自定义 制作`，如果用户给了署名则替换。
- 内容：默认从用户的“自定义内容”提炼；如果用户给出多条主题，按主题批量处理。

## 触发后的工作流

### 1. 识别任务类型

- 用户说“帮我生成图 / 绘制 / 做图 / 批量做图”：直接使用图像生成工具逐张生成。
- 用户说“给我提示词 / 批量提示词 / 生图 prompt”：只输出可复制的提示词，不生成图片。
- 用户给出一组标题、段落、清单或 CSV 式内容：每一项作为一张图，批量输出或批量生成。
- 用户没有给具体内容时，不要编造主题；给出一个简短模板，让用户填入“图像内容”和“底部标记”。

### 2. 提炼内容

每张图只表达一个核心概念。先把用户内容压缩为：

- 一个中心隐喻
- 3-6 个知识模块
- 一条观众视线流动路径
- 4-10 个短中文标注词

避免把长文原样塞进图里。中文标注应短、准、有画面感，优先使用名词短语或 2-6 字动作短语。

### 3. 构图规则

优先选择以下结构之一：

- 中心辐射：核心隐喻在中间，周围 3-6 个模块展开。
- 上下流动：从问题、机制、转化到出口。
- 左右对照：旧认知与新认知、混乱与清晰、输入与输出。
- 路径地图：用箭头、手绘线、节点和小图标形成阅读路线。
- 分层容器：把抽象系统装进盒子、岛屿、桌面、墙面、仪表盘等隐喻空间。

不要做成传统 PPT 流程图、复杂架构图、密集海报、商业模板或装饰性插画。留白必须明显，模块之间要能一眼区分。

## 固定提示词模板

生成单张图片时，用下面模板，并替换方括号内容：

```text
绘制图像，绘制一张概念性的视觉笔记信息图。

图像风格：手绘扁平矢量插画，视觉引导风格。
图像设计哲学：将复杂理念视觉综合为简单的视觉隐喻。
图像布局：有组织的模块化知识地图，干净的留白，层级流动。
图像美学：极简主义涂鸦线条艺术，搭配柔和且独特的强调色。既专业又充满趣味，注重清晰度，认知图解。
图像文字语言：中文。
图像中文字体：Ma Shan Zheng。
图像清晰度：高清。
图片大小：1242*1660像素。
图底部标记：[底部标记]。

图像内容：[图像内容]

构图要求：[中心隐喻、模块、视线流动、中文标注词]
避免：密集长文、复杂架构图、PPT模板感、商业库存插画感、过度装饰、英文文字、错别字。
```

默认底部标记是 `@自定义 制作`。

## 批量输出格式

如果用户要批量提示词，按编号输出，每条包含：

- 标题
- 核心隐喻
- 中文标注词
- 完整生图提示词

如果用户要批量生成图片，逐张调用图像生成工具，不要把多张图拼成一张。生成后简短说明每张图对应的主题和最适合的用途。

## 质量检查

生成或输出前检查：

- 是否为竖版 1242x1660。
- 是否包含中文、Ma Shan Zheng、底部署名。
- 是否把复杂理念变成视觉隐喻，而不是文字堆叠。
- 是否有清晰模块和阅读路径。
- 是否保留足够留白。
- 是否避免英文、复杂长句和密集说明。
